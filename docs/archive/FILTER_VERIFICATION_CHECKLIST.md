# 필터링 기능 검증 체크리스트

## 문제 상황
- "추천순" 선택 시 upvotes가 가장 많은 글이 최상단에 오지 않음
- 필터 변경 시 UI가 업데이트되지 않음

## 검증 체크리스트

### 1. 데이터베이스 확인
- [ ] `ideas` 테이블에 `upvotes` 컬럼이 존재하는가?
- [ ] `upvotes` 값이 실제로 저장되어 있는가? (NULL이 아닌가?)
- [ ] `upvotes` 값이 숫자 타입인가?
- [ ] 샘플 데이터의 upvotes 값 확인

### 2. Supabase 쿼리 확인
- [ ] `getIdeas` 함수에서 `sort: 'popular'`일 때 `order('upvotes', { ascending: false })`가 호출되는가?
- [ ] Supabase 쿼리가 실제로 정렬을 적용하는가?
- [ ] 네트워크 탭에서 Supabase API 응답 확인

### 3. 클라이언트 사이드 처리 확인
- [ ] `getIdeas` 반환 후 클라이언트에서 추가 정렬이 일어나는가?
- [ ] `ideas.map()` 전에 정렬이 다시 일어나는가?
- [ ] 다른 곳에서 `ideas` 배열을 수정하는가?

### 4. React 상태 업데이트 확인
- [ ] `setIdeas`가 실제로 호출되는가?
- [ ] `ideas` 상태가 실제로 변경되는가?
- [ ] React DevTools에서 상태 변경 확인

### 5. 렌더링 확인
- [ ] `ideas.map()`이 새로운 데이터로 실행되는가?
- [ ] 각 `IdeaCard`의 `key`가 변경되는가?
- [ ] React가 리렌더링을 하는가?

## 테스트 시나리오

### 시나리오 1: 추천순 정렬 테스트
1. 초기 로드 시 아이디어 목록 확인
2. "추천순" 선택
3. 첫 번째 아이디어의 upvotes 값 확인
4. 두 번째 아이디어의 upvotes 값 확인
5. 첫 번째 > 두 번째인지 확인

### 시나리오 2: 필터 변경 테스트
1. "전체 카테고리" 선택
2. "개발" 카테고리 선택
3. 아이디어 개수가 변경되는지 확인
4. 모든 아이디어의 category가 "development"인지 확인

### 시나리오 3: 서브레딧 필터 테스트
1. "전체 서브레딧" 선택
2. "r/AppIdeas" 선택
3. 아이디어 개수가 변경되는지 확인
4. 모든 아이디어의 subreddit이 "AppIdeas"인지 확인
