## 🎯 목표

관리자가 홈페이지 전반적인 수정을 자유롭고 유동적으로 관리할 수 있는 종합 관리자 시스템 구축

## 📋 확장 계획

### 1. UI/UX 관리 시스템

#### 1.1 테마 관리
- **기능**: 색상 테마, 폰트, 레이아웃 설정
- **구현 내용**:
  - 테마 색상 팔레트 설정 (Primary, Secondary, Accent 등)
  - 폰트 패밀리 및 크기 설정
  - 다크모드/라이트모드 전환 설정
  - 커스텀 CSS 주입 기능
- **예상 작업량**: 3일

#### 1.2 레이아웃 관리
- **기능**: 페이지 레이아웃, 섹션 순서 조정
- **구현 내용**:
  - 드래그 앤 드롭으로 섹션 순서 변경
  - 섹션 표시/숨김 토글
  - 그리드 레이아웃 설정 (컬럼 수, 간격 등)
- **예상 작업량**: 5일

#### 1.3 컴포넌트 스타일 관리
- **기능**: 버튼, 카드, 입력 필드 등 컴포넌트 스타일 커스터마이징
- **구현 내용**:
  - 버튼 스타일 (색상, 크기, 모서리 등)
  - 카드 스타일 (그림자, 테두리, 배경 등)
  - 입력 필드 스타일
  - 모달/다이얼로그 스타일
- **예상 작업량**: 4일

## 🗄️ 데이터베이스 스키마 확장

### 필요한 테이블

1. **`site_settings`** - 사이트 전역 설정
   - key (string, PK)
   - value (jsonb)
   - updated_at (timestamp)

2. **`ui_settings`** - UI/UX 설정
   - id (uuid, PK)
   - component_type (string) - 'button', 'card', 'header' 등
   - settings (jsonb) - 스타일 설정
   - updated_at (timestamp)

3. **`content_sections`** - 콘텐츠 섹션 관리
   - id (uuid, PK)
   - page (string) - 'home', 'community' 등
   - section_type (string)
   - content (jsonb)
   - order (integer)
   - is_visible (boolean)

4. **`advertisements`** - 광고 관리
   - id (uuid, PK)
   - name (string)
   - type (string) - 'image', 'code' 등
   - content (text)
   - position (string)
   - start_date (timestamp)
   - end_date (timestamp)
   - is_active (boolean)
   - click_count (integer)

5. **`user_roles`** - 사용자 역할
   - id (uuid, PK)
   - name (string)
   - permissions (jsonb)
   - created_at (timestamp)

6. **`user_activity_logs`** - 사용자 활동 로그
   - id (uuid, PK)
   - user_id (uuid, FK)
   - action_type (string)
   - details (jsonb)
   - created_at (timestamp)

7. **`moderation_actions`** - 모더레이션 액션
   - id (uuid, PK)
   - content_type (string) - 'post', 'comment' 등
   - content_id (uuid)
   - action_type (string) - 'approve', 'reject', 'hide' 등
   - moderator_id (uuid, FK)
   - reason (text)
   - created_at (timestamp)

## 🔧 기술 스택

- **Frontend**: React 19 + TypeScript
- **UI Components**: Shadcn/UI

- **Drag & Drop**: @dnd-kit (레이아웃 관리용)
