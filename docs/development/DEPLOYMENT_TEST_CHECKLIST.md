## ✅ 배포 후 테스트 항목

### 1. Vercel 배포 상태 확인
- [ ] Vercel Dashboard → Deployments에서 최신 배포 상태 확인
- [ ] 빌드 로그에서 에러 없음 확인
- [ ] 배포 URL 접속 가능 확인: https://ideaspark-pi.vercel.app

### 2. 개발 소식 페이지 테스트
- [ ] `/news` 경로 접속 확인
- [ ] 통합 피드가 올바르게 표시되는지 확인 (탭 없음)
- [ ] 개발 소식 카드가 최신순으로 표시되는지 확인
- [ ] 각 카드에 다음 정보가 표시되는지 확인:
  - [ ] 제목
  - [ ] 내용 (selftext 또는 selftext_html에서 추출)
  - [ ] 서브레딧 이름 (r/...)
  - [ ] 작성자
  - [ ] 수집 시간
  - [ ] 좋아요 수 (TrendingUp 아이콘)
  - [ ] 댓글 수 (MessageSquare 아이콘) ⚠️ SQL 마이그레이션 실행 후 확인
  - [ ] 태그
  - [ ] 카테고리
  - [ ] "Reddit에서 보기" 버튼

### 3. 필터링 개선 확인
- [ ] 개발 관련성 높은 콘텐츠만 표시되는지 확인
- [ ] 비개발 관련 콘텐츠(예: "나노 바나나", "여자친구 그리기")가 필터링되었는지 확인
- [ ] AI, 개발 팁, 기술 스택 관련 콘텐츠가 우선 표시되는지 확인

### 4. 주간 소식 내용 표시 확인
- [ ] 주간 소식에도 내용이 표시되는지 확인 (이전에는 제목만 표시됨)
- [ ] 내용이 HTML 태그 없이 깔끔하게 표시되는지 확인

### 5. 모바일 반응형 테스트
- [ ] 모바일 화면에서 개발 소식 페이지 접속
- [ ] 카드 레이아웃이 모바일에 최적화되어 있는지 확인
- [ ] 텍스트가 잘리지 않고 가독성이 좋은지 확인
- [ ] 버튼 크기가 터치하기 적절한지 확인

### 6. 개발 소식 수집 기능 테스트 (관리자 전용)
- [ ] 관리자 계정으로 로그인
- [ ] 개발 소식이 없을 때 "개발 소식 수집하기" 버튼 표시 확인
- [ ] 수집 버튼 클릭 시 수집 진행 확인
- [ ] 수집 완료 후 토스트 메시지 표시 확인
- [ ] 수집된 소식이 피드에 표시되는지 확인

### 7. 자동 한국어 번역 확인
- [ ] 페이지 로드 후 Google Translate 위젯이 표시되는지 확인
- [ ] 자동으로 한국어로 번역되는지 확인 (약 1초 후)
- [ ] `index.html`의 `lang="ko"` 속성이 적용되었는지 확인

### 8. 커뮤니티 버튼 스타일 확인
- [ ] 헤더의 "커뮤니티" 버튼이 다른 버튼과 스타일이 통일되었는지 확인
- [ ] 버튼이 disabled 상태가 아닌지 확인

## 🐛 알려진 이슈 및 해결 방법

### 이슈 1: 댓글 수가 0으로만 표시됨
**원인**: SQL 마이그레이션이 실행되지 않았거나, 수집 로직에서 `num_comments`를 가져오지 못함

**해결**:
1. SQL 마이그레이션 실행 확인
2. 새로운 소식 수집 후 확인
3. 기존 데이터는 `num_comments = 0`으로 유지됨 (정상)

### 이슈 2: 개발 소식이 표시되지 않음
**원인**: 데이터가 수집되지 않았거나, 필터링 기준이 너무 엄격함

**해결**:
1. 관리자로 로그인하여 수동 수집 실행
2. Supabase Dashboard에서 `dev_news` 테이블 데이터 확인
3. 필터링 점수 임계값 확인 (현재: 3점 이상)

### 이슈 3: 자동 번역이 작동하지 않음
**원인**: Google Translate 스크립트 로드 지연

**해결**:
1. 페이지 새로고침
2. 브라우저 콘솔에서 에러 확인
3. Google Translate 스크립트가 차단되지 않았는지 확인
