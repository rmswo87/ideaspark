# PRD 생성 문제 수정 보고서

**작성일**: 2025년 12월 3일  
**문제**: 생성된 PRD가 실제 아이디어와 무관한 "감정적 투쟁" 프로젝트 내용으로 생성됨

---

## 🔍 문제 분석

### 발견된 문제
- PRD 생성 시 실제 아이디어 내용을 무시하고 `prd_example.md` 파일의 예시 내용("감정적 투쟁" 프로젝트)을 그대로 사용
- AI가 프롬프트의 예시를 실제 아이디어로 오해하여 잘못된 PRD 생성

### 원인
1. 프롬프트에서 아이디어 정보가 예시보다 덜 강조됨
2. 예시와 실제 아이디어 정보의 구분이 명확하지 않음
3. "절대 금지 사항"이 충분히 강조되지 않음

---

## ✅ 수정 사항

### 1. 아이디어 정보 강조
- 아이디어 정보 섹션을 `⚠️⚠️⚠️ CRITICAL`로 강조
- "반드시 이 내용만 사용하세요" 명시
- 다른 예시나 템플릿 사용 금지 명확히 표시

### 2. 절대 금지 사항 강화
- "감정적 투쟁", "부부", "경쟁 점수" 등 구체적인 잘못된 예시 명시
- 다른 프로젝트 내용 복사 금지 명확히 표시
- 예시 파일 내용 사용 금지 명시

### 3. 필수 사항 강화
- 아이디어 내용만 분석하여 작성하도록 강조
- 아이디어의 실제 내용과 100% 일치하는 PRD만 작성하도록 명시

### 4. 예시 다이어그램 설명 개선
- 예시는 형식만 참고하도록 명시
- 실제 내용은 아이디어 정보를 분석하여 작성하도록 강조

## 🧪 테스트 필요 사항

1. **다양한 아이디어로 PRD 생성 테스트**
   - 실제 Reddit 아이디어로 PRD 생성
   - 생성된 PRD가 아이디어 내용과 일치하는지 확인

2. **예시 내용 혼입 방지 확인**
   - "감정적 투쟁" 관련 내용이 생성되지 않는지 확인
   - 다른 예시 프로젝트 내용이 혼입되지 않는지 확인

3. **아이디어 내용 정확도 확인**
   - 생성된 PRD의 프로젝트명이 아이디어 제목과 관련 있는지 확인
   - 기능 목록이 아이디어 내용과 일치하는지 확인
