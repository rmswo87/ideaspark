## 🎨 Magic MCP 활용 전략

### 단계별 접근
1. **Inspiration 단계**: 각 옵션의 참고 컴포넌트 검색
2. **Builder 단계**: 선택한 옵션의 컴포넌트 생성
3. **Refiner 단계**: 기존 컴포넌트와 통합 및 미세 조정

### 예시 워크플로우
```typescript
// 1. 참고 컴포넌트 검색
mcp_Magic_MCP_21st_magic_component_inspiration({
  query: "minimalist dashboard",
  registries: ["@shadcn"]
})

// 2. 컴포넌트 생성
mcp_Magic_MCP_21st_magic_component_builder({
  message: "미니멀리스트 대시보드 레이아웃",
  searchQuery: "minimalist dashboard",
  // ...
})

// 3. 기존 컴포넌트 개선
mcp_Magic_MCP_21st_magic_component_refiner({
  userMessage: "더 깔끔하고 모던한 느낌으로 개선",
  absolutePathToRefiningFile: "src/App.tsx",
  context: "메인 대시보드 레이아웃"
})
```

## 📊 선택 가이드

### 빠른 결정을 위한 체크리스트

1. **주요 사용자 타겟은?**
   - 개발자/기술자 → 조합 1
   - 비즈니스 사용자 → 조합 2
   - 크리에이티브 사용자 → 조합 3

2. **성능이 최우선인가?**
   - 예 → 조합 1
   - 아니오 → 조합 2 또는 3

3. **모바일 사용자 비중은?**
   - 높음 → 조합 1
   - 중간 → 조합 2
   - 낮음 → 조합 3

4. **개발 시간은?**
   - 짧음 (5일 이내) → 조합 1
   - 중간 (5-7일) → 조합 2
   - 여유 있음 (7일 이상) → 조합 3
