1. **비즈니스 문의**
   - 실제 Supabase `contact_inquiries` 테이블 사용
   - 마이그레이션만 적용하면 즉시 작동
   - 이메일 알림은 선택사항 (추가 구현 시 Resend 등 사용)

2. **도네이션**
   - 실제 계좌번호 사용
   - QR 코드 이미지 실제 파일 사용
   - Mock 데이터 없음

3. **모든 다른 기능**
   - Reddit API: 실제 API 사용
   - AI 생성: 실제 OpenRouter API 사용
   - 사용자 데이터: 실제 Supabase 사용
   - 파일 저장: 실제 Supabase Storage 사용

### ⚠️ 추가 설정이 필요한 구간

#### 1. 비즈니스 문의 테이블 생성
**파일**: `supabase/migrations/20250129_create_contact_inquiries_table.sql`  
**설정 방법**:
1. Supabase Dashboard 접속
2. SQL Editor 열기
3. 마이그레이션 파일 내용 복사하여 실행
4. 또는 Supabase CLI 사용: `supabase migration up`

**설정 후**: 문의 내용이 자동으로 Supabase에 저장됨

#### 2. 관리자 문의 확인 기능 (선택사항)
**현재 상태**: 문의는 저장되지만 관리자가 확인할 UI 없음  
**구현 필요**:
- 관리자 대시보드에 문의 목록 탭 추가
- 문의 상세 보기 및 답변 기능
- 상태 관리 (pending, replied, closed)

**예상 작업량**: 1-2일

#### 3. 이메일 알림 (선택사항)
**현재 상태**: 문의 저장만 됨, 이메일 알림 없음  
**구현 방법**:
1. Resend 계정 생성 (https://resend.com)
2. API 키 발급
3. Supabase Edge Function 생성
4. 문의 저장 시 Edge Function 호출하여 이메일 전송

**예상 작업량**: 2-3일
